# MedSim Frontend

Frontend-часть платформы `MedSim` для учебной симуляции процессов медицинской организации.

## Что уже реализовано

Сейчас фронт покрывает первый пользовательский сценарий:

- игрок вводит имя;
- выбирает свою реальную должность в больнице;
- вводит код игровой сессии;
- подключается к backend;
- после успешного входа попадает в стартовую комнату ожидания.

Также на стартовой странице уже есть отдельный блок входа для:

- ведущего;
- суперпользователя.

Пока служебный вход только сверстан и не подключён к backend-авторизации.

## Технологии

- `React`
- `Vite`
- `TypeScript`
- CSS без UI-библиотеки

## Визуальный подход

Текущий интерфейс выдержан в минималистичном зелёном стиле, чтобы:

- визуально ассоциироваться с медицинской средой;
- не перегружать стартовый экран;
- сохранить единый стиль между формой входа и комнатой ожидания.

## Архитектурная идея

Структура фронта уже разбита по ролям ответственности:

- `components/`
  Переиспользуемые визуальные части интерфейса.
- `constants/`
  Константы, которые не должны жить внутри JSX.
- `hooks/`
  Состояние и сценарная логика.
- `services/`
  Работа с backend API.
- `types/`
  Общие типы приложения, форм и ответов сервера.

Это сделано специально, чтобы дальше было проще добавлять:

- экран ведущего;
- роутинг;
- новые API-сценарии;
- страницы по ролям.

Переход на `TypeScript` даёт ещё и более надёжную работу с:

- формами;
- ролями;
- API-ответами backend;
- дальнейшим рефакторингом приложения.

## Backend API

По умолчанию фронт обращается к:

- `http://localhost:8080`

Это настраивается в:

- `src/services/playerSessionsApi.ts`

Если понадобится другой backend URL, можно добавить переменную окружения:

```env
VITE_API_BASE_URL=http://localhost:8080
```

## Текущий пользовательский поток

### Игрок

1. Открывает стартовую страницу.
2. Вводит:
   - имя;
   - должность;
   - код сессии.
3. После успешной отправки формы попадает в стартовую комнату.

На экране ожидания сейчас отображаются:

- код сессии;
- имя участника;
- реальная должность;
- игровая роль, если она уже назначена;
- статус сессии.

### Ведущий / суперпользователь

Для них уже есть отдельный UI:

- выбор типа доступа;
- логин;
- пароль.

Но сама серверная авторизация для этих ролей ещё впереди.
